## Ways to Contribute

You can contribute by:

- Reporting bugs  
- Suggesting improvements  
- Proposing new features  
- Improving documentation  
- Contributing code  
- Sharing case studies or applications  
- Supporting teaching and training material  

All contributions help strengthen the project.

---

## Reporting Issues

If you encounter a bug or unexpected behaviour:

1. Check if the issue has already been reported.
2. Open a new issue in the relevant repository under the  
   [OnSSET GitHub organisation](https://github.com/orgs/OnSSET/repositories).

Please include:

- Model version  
- Input description  
- Error message or observed behaviour  
- Steps to reproduce  

Clear reports help maintainers respond efficiently.

---

## Proposing New Features

For new functionality or methodological extensions:

- Describe the problem the feature addresses  
- Explain the proposed solution  
- Provide use cases or examples  
- Indicate whether it affects model assumptions or outputs  

Feature discussions can start in GitHub Discussions or the modelling forum.

---

## Contributing Code

If you would like to contribute code:

1. Fork the relevant repository  
2. Create a new branch  
3. Implement your changes  
4. Submit a pull request  

We encourage:

- Clear commit messages  
- Descriptive pull request summaries  
- Documentation updates when relevant  
- Backward compatibility where possible  

Pull requests may be reviewed before merging.

---

## Development Principles

OnSSET development follows:

- Transparent version control  
- Reproducible workflows  
- Documented assumptions  
- Open discussion of methodological choices  

The project aligns with the U4RIA principles for open energy modelling.

---

## Recognition

Contributors are acknowledged through:

- GitHub contribution history  
- Release notes  
- Steering Committee discussions where relevant  

We value collaborative and respectful engagement.
